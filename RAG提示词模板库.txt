                    VCPBookIndexGen - RAG知识库提示词模板库
================================================================================

【对话总结】

用户询问本项目能否处理知识类、教学类书籍（而非仅小说）。
经分析代码，本项目核心逻辑（章节提取、文本分块、LLM调用）与内容类型无关，
只需修改 summarizer.py 中的 prompt 模板即可适配不同类型书籍。

关键结论：
1. 章节匹配支持自定义正则（--pattern 参数），可适配各种格式
2. 所有 prompt 集中在文件顶部，修改方便
3. 改知识类书籍只需替换"故事归纳"为"知识提炼"，调整输出维度
4. 可改为问答对形式，更适合RAG检索

================================================================================
                         提示词模板设计原则（RAG优化）
================================================================================

RAG知识库的核心需求：
1. 结构化输出 - 便于向量检索和语义匹配
2. 关键信息提取 - 减少噪音，提高检索精度
3. 问答对形式 - 直接匹配用户查询意图
4. 实体与关系 - 支持知识图谱构建
5. 元数据标注 - 便于过滤和分类

================================================================================
                              使用说明
================================================================================

【如何选择模板】

1. 小说/文学作品 → 模板一（剧情人物设定）
2. 教材/科普/技术文档 → 模板二（知识点体系）或 模板三（问答对）
3. 历史著作 → 模板四（时间线人物事件）
4. 写作教学 → 模板五（理论技法案例）
5. 周易/哲学 → 模板六（概念象征应用）
6. 编程教材 → 模板七（语法接口代码）
7. 商业管理 → 模板八（理论方法案例）

【RAG优化建议】

1. 优先使用模板三（问答对形式），检索效果最佳
2. 可组合使用：先用模板二提取知识点，再用模板三生成问答
3. 输出时保留章节标题作为元数据，便于溯源
4. 对于超长章节，步进式总结会自动处理

【修改方法】

打开 novel_indexer/summarizer.py 文件，将对应模板复制到文件顶部的常量定义处：
- PROMPT_CHAPTER_SUMMARY → 单章节总结
- PROMPT_BOOK_SUMMARY → 全书总结
- PROMPT_PROGRESSIVE → 长章节步进总结（可选修改）

================================================================================
                    章节识别正则表达式参考表（--pattern 参数）
================================================================================

【什么是正则表达式？】

正则表达式（Regular Expression，简称 regex）是一种用来"匹配文本模式"的工具。
简单说，你告诉程序"章节标题长什么样"，程序就能把所有章节找出来。

【程序内置的自动识别】

程序默认会自动尝试以下模式，大多数情况下不用你操心：
- 第X章、第X节、第X回（支持中文数字和阿拉伯数字）
- Chapter 1、Chapter 2...（英文书籍）
- 1.、2.、3.（纯数字编号）

只有当自动识别失败或不准确时，才需要用 --pattern 参数手动指定。

================================================================================
                         正则表达式基础语法（小白必读）
================================================================================

【最常用的几个符号】

符号        含义                    举例
---------------------------------------------------------------------------
^           行的开头                ^第 表示"以'第'开头的行"
\d          任意数字（0-9）         \d+ 表示"一个或多个数字"
+           前面的内容出现1次或多次  \d+ 匹配 "1"、"123"、"9999"
*           前面的内容出现0次或多次  \d* 匹配 ""、"1"、"123"
?           前面的内容出现0次或1次   第?\d+ 匹配 "第1" 或 "1"
\.          真正的点号（.有特殊含义）  \. 匹配文本中的 "."
\s          空格或制表符            \s+ 匹配一个或多个空格
.*          任意内容                第.*章 匹配 "第一章"、"第1章 标题"
[...]       匹配括号里的任意一个     [一二三] 匹配 "一" 或 "二" 或 "三"
|           或者                    第一章|第二章 匹配 "第一章" 或 "第二章"

【中文数字匹配】

程序已内置中文数字识别，你不用自己写。但如果需要，可以这样：

[\d零一二三四五六七八九十百千万]+  匹配中文或阿拉伯数字

================================================================================
                         --pattern 参数使用教程
================================================================================

【基本用法】

在命令行运行程序时，加上 --pattern 参数：

python main.py 书籍文件.txt --pattern "正则表达式"

【完整示例】

示例1：识别 "1.1 标题" 这种格式
python main.py 教材.txt --pattern "^\d+\.\d+"

示例2：识别 "第一章 标题" 这种格式
python main.py 小说.txt --pattern "^第[\d零一二三四五六七八九十百千万]+章"

示例3：识别 "Chapter 1" 英文格式
python main.py book.txt --pattern "^[Cc]hapter\s+\d+"

示例4：识别 "【第一章】标题" 带括号格式
python main.py 书籍.txt --pattern "^【第[\d零一二三四五六七八九十百千万]+章】"

【注意事项】

1. 正则表达式要用英文双引号包裹
2. 如果正则里有特殊字符，可能需要转义
3. 程序会先让你预览识别结果，确认无误再继续
4. 如果识别不准，可以按 r 重新输入正则

【交互流程】

运行程序后会显示预览：
┌──────────────────────────────────────┐
│ #    标题                    行号    │
├──────────────────────────────────────┤
│ 1    第一章 开始            15      │
│ 2    第二章 相遇            234     │
│ ...                                  │
└──────────────────────────────────────┘

操作选择: [y]确认 / [r]输入新正则重试 / [m]手动编辑目录文件 / [q]退出

- y: 确认识别正确，继续处理
- r: 正则不对，重新输入
- m: 自动识别完全失败，手动编辑目录文件
- q: 放弃处理

                         常见问题解答
================================================================================

Q: 我的书章节标题是 "一、xxx" "二、xxx" 这种，怎么写正则？
A: 使用 ^[\d零一二三四五六七八九十百千万]+、
   命令: python main.py 书.txt --pattern "^[\d零一二三四五六七八九十百千万]+、"

Q: 程序识别出了太多"假章节"，怎么办？
A: 可能是正则太宽松了。尝试让正则更精确：
   - 加上 ^ 确保匹配行首
   - 限制数字范围，比如 \d{1,3} 只匹配1-3位数字
   - 加上必须包含的内容

Q: 程序一个章节都没识别出来？
A: 检查以下几点：
   1. 用记事本打开书籍，看看章节标题实际长什么样
   2. 确认标题是否在行首（没有空格缩进）
   3. 如果有空格缩进，正则改成 ^\s*第\d+章

Q: 正则表达式太复杂了，我学不会怎么办？
A: 没关系！你可以：
   1. 先运行程序，让它自动识别试试
   2. 如果识别不准，按 m 选择手动编辑目录文件
   3. 手动在文件里列出所有章节标题，程序会按你的列表切分

Q: 我有多个正则需要同时匹配怎么办？
A: 用 | 符号连接，比如：
   ^(序章|第\d+章|尾声)
   这会同时匹配 "序章"、"第1章"、"第2章"...、"尾声"
